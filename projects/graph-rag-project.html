Graph-based RAG for Neuroscience Research

Built a knowledge graph-driven retrieval system to answer complex neuroscience questions with paragraph-level evidence and citations.

Python Neo4j Graph RAG LLMs (LLaMA-70B) NLP / NER

Problem & Context

Traditional vector-based RAG systems work well for simple lookups but struggle with complex, multi-hop neuroscience queries. They often rely on limited context, single-source retrieval, and lack clear traceability to original research papers. This makes it difficult to generate reliable, evidence-backed answers in a domain where accuracy and citations are critical.

My Role & Scope

  • Independently designed and built the complete data pipeline for a Graph-based RAG system
  • Scraped and structured neuroscience research papers from PMC in a paragraph-level hierarchy
  • Designed prompt-based pipelines using LLaMA-70B for claims, species, entity, and relationship extraction
  • Constructed and populated a Neo4j knowledge graph with entity and paragraph nodes
  • Enabled graph-based retrieval with explicit provenance and paragraph-level citations
  • Scaled large-scale information extraction by implementing Python multiprocessing, enabling parallel processing of hundreds of research papers during LLM-based extraction workflows.

Architecture & Approach

  • Scraped neuroscience research papers (XML) from PMC and organized them into a structured filesystem
  • Extracted claims, contributions, and experimental species using prompt-engineered LLaMA-70B workflows
  • Performed domain-specific NER using LexiFuzz with NIF ontology at paragraph level
  • Extracted semantic relationships between entity pairs using LLMs and stored them as structured JSON
  • Injected entities and relationships into Neo4j, linking entities to exact source paragraphs
  • Answered user queries by identifying relevant entities, executing Cypher queries, and grounding responses in original text

Key Technical Decisions

  • Modeled the graph with two node types (Entity and Paragraph) to preserve fine-grained provenance and traceability.
  • Used RELATED_TO relationships between entities and DESCRIBES edges to link entities to source paragraphs.
  • Chose paragraph-level processing instead of document-level extraction to improve accuracy and reduce hallucinations.
  • Combined ontology-based NER with fuzzy matching to balance precision and recall in a specialized domain.
  • Implemented a high-throughput multiprocessing pipeline to process thousands of research papers efficiently. By running up to 100 parallel processes, each handling independent LLaMA-70B inference, the system significantly reduced end-to-end extraction time.

Results & Impact

  • Enabled multi-hop, graph-based reasoning for complex neuroscience queries
  • Generated answers with explicit paragraph-level evidence and PMC references
  • Outperformed bio-specific LLMs (Bio-Mistral, SciLitLLM) in depth, rigor, and citation quality
  • Achieved significantly higher evaluation scores in zero-shot comparisons against GPT-4o
