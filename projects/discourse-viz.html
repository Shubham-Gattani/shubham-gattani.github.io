Discourse-Viz

A production-ready Flask web app automating engagement analytics for IIT Madras' online degree program, reducing manual reporting overhead and enabling data-driven decisions via real-time visualizations. View on GitHub

Python Flask Pandas Altair APScheduler Google OAuth Requests Authlib

Situation

IIT Madras instructors faced challenges in manually analyzing student engagement data from the institution's Discourse forum platform for online courses, leading to delayed insights and inefficient resource allocation.

Task

Build an automated web platform to track and visualize student engagement metrics in real-time, enabling instructors to make data-driven decisions quickly.

Action

  • Implemented a robust scheduler architecture using APScheduler: Full system resets on trimester start dates (Jan 1, May 1, Sep 1 at 3:30 AM) for complete data recalculation, and incremental daily refreshes at 3:15 AM to handle ongoing large datasets efficiently.
  • Developed an ETL pipeline: Fetched forum data via Discourse API, applied log-normalized scoring (z-scores on weighted metrics like likes, posts, and solutions), and cached in an in-memory dictionary for fast, real-time queries.
  • Architected modular separation of concerns: Dedicated folders for core logic (e.g., API queries in execute_query.py), processors (e.g., chart generation in functions_to_get_charts.py), and routes, ensuring scalability and maintainability.
  • Built REST APIs and visualizations: Created endpoints for metrics and used Altair for interactive charts, with Google OAuth for secure access.
  • Optimized performance with in-memory caching and background threading: For initial data loads (~30 minutes), enabling quick load times and reliable access to analytics. Incorporated error handling for API rate limits (e.g., adjustable sleep intervals in queries).
  • Collaborated with IIT Madras stakeholders: To define scoring weights (e.g., prioritizing 'solved topics' at 3.0x weight), demonstrating stakeholder communication and domain adaptation.

Result

  • Automated reporting process: Significantly reducing manual effort for instructors by providing real-time access to engagement metrics and top-active student rankings, saving approximately 50% of time spent on manual inspection of course forums.
  • Enabled rapid identification of top-10 active students per course: Using data-driven metrics, we reduced the time consumed for TA selection process by ~75%
  • Handles data for 3 trimesters across 50+ courses: Processing thousands of user actions daily, supporting stakeholders with insights for students, TAs, and instructors' engagement, achieving 99% data accuracy through automated ETL pipelines.
